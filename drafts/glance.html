This page provides a paraphrased summary of the Web Sustainability Guidelines (WSG) 1.0. For the normative technical specification, see https://w3c.github.io/sustyweb/.

It aims to be a helpful entry point for individuals and stakeholders who have an interest in digital sustainability and the work undertaken by the Sustainable Web Design Community Group.

While this document does provide a quick summary of numerous key aspects of the WSG specification, it must be established that it in no way should be used as a replacement for following the guidelines and success criteria as laid out in the complete, long-form specification. Reasons for this include that this summary is neither intended to be in-depth nor feature-complete. Therefore, its role as an overview to help orientate stakeholders to the specification's purpose in no way provides conformance to Web Sustainability criteria.

Introduction

Context

The digital industry is now responsible for between 2-5% of global emissions [[ICT-IMPACT]] [[EUC]] [[8BT]], more than that of the aviation industry. Since the Paris Agreement, average web page sizes have increased by over 70% on desktop and 140% on mobile [[PAGE-WEIGHT]]. Between 2015 and 2021, internet visitors increased by 60%, whilst internet traffic increased by 440% [[DATA]]. In essence, if the Internet were a country it would be one of the top five polluters. [[COUNTRY]]

Summary

The Web Sustainability Guidelines (WSG) 1.0 explains how to design and implement digital products and services that put people and the planet first. The guidelines are best practices based on measurable, evidence-based research; aimed at end-users, website creators, product owners, stakeholders, tool authors, educators, and policymakers. The guidelines are in line with both the Sustainable Web Manifesto [[MANIFESTO]] and Manifesto for a Humane Web [[HUMANE]].

Overview

The guidelines should be seen as a starting point in a sustainability journey; they cover the following:

User Experience Design

Covering research and ideation, journey design, content and assets, and quality assurance.

Show / Hide available guidance.

In relation to research and ideation, make sure you:

  • Map all areas where your digital product could impact people and the planet.
  • Assess all stakeholder issues as well as visitor and non-visitor needs; then define appropriate test policies.
  • Consider the use of low-carbon tools during ideation.

In relation to journey design, make sure you:

  • Create a frictionless, lightweight experience, using recognized design patterns that ensure navigation and way-finding are well-structured, the visitor's attention is respected and visitors aren't manipulated.
  • Implement forms that are usable, accessible, and minimal and assist visitors with errors.
  • Only implement useful notifications that improve the visitor's experience.

In relation to content and assets, make sure you:

  • Avoid unnecessary assets, but when they are needed, take a sustainable approach to images, media, animations, typefaces, and documents.
  • Provide alternatives to visual and audio content, as well as facilitate speech-based browsing of your content.
  • Write with purpose, in an accessible, easy-to-understand format.

In relation to quality assurance, make sure you:

  • Have open working and documentation processes such as design systems.
  • Conduct regular audits and testing, including value, usability, and compatibility testing into releases, as well as using inputs from visitor journey tracking to identify issues / opportunities.

Web Development

Covering development approach, code minimization, code coherence, and code security.

Show / Hide available guidance.

In relation to the development approach, make sure you:

  • Identify indicators and metrics that will encourage sustainable development approaches, and use sustainability as a success criterion of your technical requirements.

In relation to code minimization, make sure you:

  • Focus on removing bottlenecks and reducing compute load on a visitor's device by removing rendering blockages, using code splitting, managing dependencies, taking advantage of native features, rigorously assessing third-party code / services, and minifying everything.
  • Hunt out valueless code by tree shaking your code, avoiding duplication, fewer and simpler queries, and (sparingly) using the latest JavaScript and APIs.

In relation to code coherence, make sure you:

  • Enable other tools to interpret your code by using HTML and metadata correctly, adhering to accessibility standards, enabling code-based way-finding, avoiding deprecated or proprietary code, providing expected files, and using the latest standards in relation to website assets.
  • Facilitate your visitor's low-carbon choices by using a device-adaptable layout and adapting to user preferences.

In relation to code security, make sure you:

  • Use the latest stable version of your chosen code language and regularly ensure your scripts are secure.

Hosting, Infrastructure and Systems

Covering environment commissioning, minimizing environment and data, and minimizing human disruption.

Show / Hide available guidance.

Make sure you:

  • Commission environments by selecting a sustainable hosting provider, ensuring support of asynchronous processing, using the lowest appropriate infrastructure tier, and minimizing the use of additional environments.
  • Minimize environments and reduce data transfer by compressing files, managing duplicate data, storing data according to visitor needs, optimizing caching, refreshing data only when the visitor needs it, and using CDNs and edge caching.
  • Reduce human disruption by automating where appropriate and dealing with errors and redirects.

Business Strategy and Product Management

Covering reporting, disclosure, strategy, and policies from both an organizational and website / product level.

Show / Hide available guidance.

In relation to organizational level reporting and disclosure, make sure you:

  • Assign responsibility for sustainability as a core part of people's roles.
  • Define clear sustainability goals and metrics, which are verified and upon which you report regularly so that others can follow in your footsteps.

In relation to organizational level strategy and policies, make sure you:

  • Have business models based on positive impact, which actively listen to all stakeholders, and ensure economic benefits are shared with employees, suppliers, and wider society.
  • Define and implement policies for Justice, Equity, Diversity and Inclusion, ethical and responsible finance / investments, philanthropy, working with suppliers who align with your values, reduction of e-waste, data privacy, appropriate data management, and considered use of emerging technologies.
  • Fully induct your employees on all of the above.

In relation to product or website-level strategy policies, make sure you:

  • Establish if the product or website is needed; if it is, then define its requirements, design an ethical and sustainable product strategy to guide it, estimate its emissions, put budgets around those emissions, and communicate to visitors the impact their decisions will have.
  • Once live, follow a product management and maintenance strategy to keep efficient, continuously improve it, openly and honestly communicate the changes you've made, and plan for its end of life.
  • Use open source software.
